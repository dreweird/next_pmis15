Title: Replace legacy `url.parse()` usage with WHATWG `URL` API

Description:

I found a deprecation warning caused by `url.parse()` when running the app in production. The project uses `oauth4webapi` (v3.8.3) transitively via `next-auth` / `@auth/core`. The package includes a fallback that uses `URL.parse` when available; this triggers Node's `DEP0169` deprecation warning.

Proposed change (small):
- Replace the conditional call to `URL.parse` with the WHATWG `URL` constructor only. If parsing fails, return `null` as the current fallback behavior.

Patch file:
- See `patches/oauth4webapi+3.8.3.patch` in this repository for the full patch generated by `patch-package`.

Suggested PR body:
- Why: Using Node's legacy `url.parse()` causes deprecation warnings. The WHATWG `URL` API is the recommended approach and avoids the deprecation.
- What: Replace `const URLParse = URL.parse ? (url, base) => URL.parse(url, base) : (url, base) => { try { return new URL(url, base) } catch { return null }};` with a single fallback that uses `new URL(url, base)`.
- Backwards compatibility: The fallback remains tolerant (returns `null` on parse failure). Tests should be added/verified.
